Результат командной работы

frontend

Deploy
  1. Адаптипная верстка
  2. Единый стиль приложения, реализованный с помощью tailwindcss
  3. Регистрация
    1. При нажатии на кнопку "create account" происходит открытие формы регистрации
    2. После валидации открывается страница с рабочими пространствами
    3. Есть валидация данных
  4. Авторизация/logout
    1. После нажатия на кнопку "Already have an account?" открывается формы для авторизации
    2. Пользователь должен ввести данные, которые были указаны при регистрации
    3. Есть валидация данных
  5. Смена языка (русский, английский, украинский)
  6. Смена темы
    1. Реализованы три цветовые схемы
    2. Изменяются так же визуальные стили для интерактивных элементов
    3. Изменение тем затрагивает в том числе svg-изображения
  7. Создание досок
    1. Избранные доски Реализовано отображение избранных досок, не реализовано добавление/удаление
    2. Карточки только добавление без редактирования и удаления
  1. Колонки (добавление, редактирование удаление)
  2. Drag&Drop
  3. Поиск по доскам
  4. Взаимодействие с другими пользователями

backend

  1. Использован Rest API
  2. Использован паттерн MVC
  3. Используется база данных Mongo DB
  4. Используется ORM Mongoose
  5. Для приложения использовались nodejs и express
  6. Сервер корректно работает с пользователем:
    1. добавление пользователя
    2. удаление пользователя
    3. изменение данных пользователя
    4. получение одного/всех пользователя(ей)
  7. Сервер корректно работает с доской:
    1. добавление доски
    2. удаление доски
    3. изменение доски
    4. получение доски/ всех досок
  8. Сервер корректно работает с карточками:
    1. добавление карточки
    2. удаление карточки
    3. изменение карточки:
    4. получение конкретной карточки/всех карточек
  9. Реализованы связи между элементами БД:
    1. связь пользователя с доской и карточкой по id (создатель)
    2. связь доски с пользователем-создателем и пользователями-участниками по id
    3. связь карточки с пользователем-создателем, пользователями/участниками, а также с доской, в которой она создается:
  10. В случае ошибок приложение возвращает код ошибки с кратким описанием проблемы
